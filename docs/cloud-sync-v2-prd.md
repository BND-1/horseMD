# 云同步 v2 PRD

> 状态：实施中
> 更新时间：2026-07-19
> 范围：桌面端 WebDAV / S3 文件夹同步的方向选择、远端重置保护与可恢复变更。

## 1. 背景与问题

v1 用本地快照和远端 manifest 推导双向变更。它能处理常规的新增、下载、冲突和受控删除，但把“远端工作区被手工清空”误判为来自另一台设备的删除。因此可能生成删除本地文件的计划，既不符合用户意图，也不满足数据安全要求。

成熟系统提供了可借鉴的边界：Syncthing 区分发送并接收、仅发送、仅接收；Joplin 也说明空同步目标会被解释为删除；Obsidian Sync 依赖版本历史恢复。HorseMD v2 不复制任何一个产品的全部实现，而采用它们共同的安全原则：方向明确、删除可恢复、未知远端状态不自动执行。

## 2. 目标与非目标

### 目标

1. 用户能明确选择“上传本地到云端”或“从云端下载到本地”。
2. 已建立基线的工作区仍可手动执行普通双向同步。
3. 云端工作区标记或 manifest 消失时，绝不自动删除本地文件。
4. 方向化覆盖、删除和冲突都保留被替代的文件到本地或远端回收路径。
5. 保持普通 Markdown、图片和附件的原有相对路径；不将资料迁入私有数据库。

### 非目标

- 不在 v2 做实时协同、自动文本三方合并或 E2EE。
- 不把云端对象存储扫描结果当作可信删除日志。
- 不支持移动端同步入口，直到移动端安全存储和网络实现完成。

## 3. 用户模型

### 3.1 三种操作

| 操作 | 用户含义 | 结果 |
| --- | --- | --- |
| 上传本地到云端 | 本地文件夹是正确来源 | 上传本地内容；远端多余/被替代文件先进入远端回收路径 |
| 从云端下载到本地 | 云端工作区是正确来源 | 下载远端内容；本地多余/被替代文件先进入本地 `.horsemd/trash/` |
| 双向同步 | 已初始化的多设备日常使用 | 仅本地改则上传，仅云端改则下载，同文件双方改则生成冲突副本 |

“上传”和“下载”是一次性的**恢复/初始化动作**，不是隐藏的永久偏好。成功后会重新建立本机基线；用户可再使用双向同步。

### 3.2 首次和第二台设备

1. 第一台设备：创建或选择空远端工作区，选择“上传本地到云端”。
2. 第二台设备：选择已有远端工作区，选择“从云端下载到本地”。
3. 两端都有独立文件时：明确选择“合并两个已有文件夹”（高级操作），预览后才执行。

### 3.3 远端被清空或损坏

当本机存在成功同步基线，但远端 manifest 缺失，或远端文件清单被异常清空且没有对应墓碑时，显示：

```text
云端工作区可能被清空或被替换
本地文件没有被修改。

[上传本地文件以恢复云端] [选择其他云端工作区] [取消]
```

双向同步在该状态下被阻止。用户若确实希望本地变为空，只能显式选择“从云端下载到本地”，并查看受保护的删除清单。

## 4. 数据安全规则

1. 无用户明确方向时，不执行方向化覆盖或删除。
2. 执行上传前，远端同名旧版本写入 `.horsemd/trash/<timestamp>/`；远端额外文件也先归档再删除。
3. 执行下载前，本地同名旧版本或本地额外文件移动到 `.horsemd/trash/<timestamp>/`。
4. 双向同步中只有 manifest 的墓碑才是可信删除事件。外部删除 manifest、清空 Bucket、手工改动对象都属于“需要恢复决策”。
5. 双向冲突不做静默覆盖。第一版保留双方：标准路径保留远端版本，本地版本创建带来源的冲突副本并上传。
6. 所有预览都列出文件级操作；方向化操作和含删除/冲突的双向操作必须确认。

## 5. 状态机

```text
未绑定连接
  -> 已绑定，未建立基线
    -> 上传本地 / 下载云端 / 高级合并
      -> 已建立基线
        -> 双向同步

已建立基线 + 远端 manifest 缺失或异常清空
  -> 远端重置待处理
    -> 上传本地恢复云端 / 下载云端到本地 / 取消
```

`远端重置待处理`不是失败后的自动重试状态，必须由用户选择恢复方向。

## 6. 验收标准

1. 空远端 + 上传本地：所有本地 Markdown、图片、附件上传，且本机状态建立。
2. 空本地 + 下载云端：所有远端内容下载，且本机状态建立。
3. 双端各有文件：双向预览不静默覆盖；冲突副本保留。
4. 清空远端对象或删除 manifest：普通双向预览返回 `remote-reset`，不产生 `deleteLocal` 操作。
5. 选择上传恢复：本地重新写入远端，不因旧本机快照产生删除本地操作。
6. 选择下载：本地被替换或多余文件均可在 `.horsemd/trash/` 找回。
7. WebDAV 和 S3/MinIO 均覆盖上述链路；中断、认证失败、条件写失败不得推进本机基线。
