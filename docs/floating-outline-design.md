# 悬浮章节导航设计

## 目标

为长 Markdown 文档提供不打扰写作的章节定位。默认仅在编辑区右侧显示一列小圆点；鼠标进入右侧导航区域后展开章节名称，点击可跳转到对应标题。

它是现有左侧“大纲”面板的轻量阅读入口，不替代文件树或完整大纲的折叠、拖拽重排能力。

## 范围与边界

- 仅桌面端显示；移动端保持现有抽屉大纲，避免遮挡触摸编辑和系统手势。
- 仅在普通文档有至少一个标题时显示；欢迎页、设置页、无标题文档不渲染。
- 左侧侧栏处于“大纲”模式时仍显示；完整大纲用于管理和重排，右侧悬浮导航用于阅读时快速定位，二者分别服务不同操作。
- 分屏时仅服务当前聚焦的文档，固定在编辑中心区域右侧，不覆盖另一侧文档。
- 富文本与源码模式共用 `useOutline` 的标题列表、当前章节索引和 `jumpToHeading`；不新增第二套 heading 扫描或 scroll listener。
- 大型文档继续使用 `useOutline` 的缓存标题 offset scrollspy。悬浮组件只能渲染状态，不能在每次滚动时读取布局。

## 交互

### 收起态

- 右侧显示 4–8 个等距圆点。标题较多时圆点按章节顺序采样，首尾和当前章节始终可见。
- 当前章节是实心强调点，普通项为细边点；鼠标悬停显示该点关联标题的原生 tooltip。
- 当前章节变化只更新强调状态，不触发列表重排。

### 展开态

- 鼠标进入圆点或导航区域后，在 180ms 内向左展开成紧凑列表；离开后 220ms 收起。
- 显示全部标题，按照标题层级缩进。列表有固定最大高度，超出时在内部滚动，不影响文档页面。
- 当前项使用现有主题 accent 的浅色背景和强调文字，不新增发光、渐变或大面积浮层。
- 标题默认单行省略；当前标题最多两行，以便阅读。所有条目保留完整 `title` 属性。
- 点击任意条目调用既有 `jumpToHeading`，沿用其平滑动画、异步内容稳定和 overflow-anchor 保护。

### 可访问性

- 圆点和章节条目都是真实 `button`，带 `aria-label`、当前项 `aria-current` 与键盘 focus-visible 样式。
- 目录容器是导航 landmark，名称为“文档目录”。

## 模块接口

`components/FloatingOutline.jsx` 是无状态展示组件：

```js
{
  headings: [{ level, text }],
  activeIndex: number,
  onJump: (index) => void
}
```

可见性、模式判断、滚动状态均留在 `App.jsx` 与 `useOutline.js`，不把编辑器 ref、DOM 查询或滚动逻辑放入组件。

## 验收

1. 有多级标题的桌面文档显示圆点，当前滚动章节同步变为强调点。
2. 悬停展开列表，H2/H3 有层级缩进；长标题不撑开面板，完整标题可通过 tooltip 读取。
3. 点击条目后跳至正确标题，源码和富文本模式均可用。
4. 左侧大纲打开时右侧悬浮导航仍可用；移动端、设置页和无标题文档不显示。
5. 大于 12 个标题时，收起态最多显示 8 个点且保留当前章节；展开态内部可滚动。
6. 不增加滚动时逐标题 `getBoundingClientRect()` 调用；沿用 `useOutline` 缓存。
