# 表格

<span class="version-badge">适用于 HorseMD v0.6.5</span>

## 插入表格

在空段落输入 `/表格`、`/table` 或拼音首字母 `/bg`，选择表格命令。默认会插入一个可继续扩展的表格。也可以粘贴标准 GFM 表格源码。

点击单元格开始输入，使用表格边缘的控制按钮添加或删除行列。长文本和行内代码会在列宽内换行，不会覆盖相邻单元格。

## 宽表和列宽

内容较少的表格会按内容保持自然宽度，不会自动拉满整条编辑区。列数较多或单元格内容较长时，只有表格自身出现横向滚动条，正文页面不会被撑宽。

将鼠标移到某一列的边界，先出现的是加列操作。若要调整列宽，请在边界处按住约一小会儿后再左右拖动；列宽会跟随鼠标实时变化，松开鼠标后保留这次宽度。普通点击和短暂悬停不会改变列宽。

在宽表最右侧进行加列、列菜单或列宽调整时，表格会停留在当前横向位置，不会自动跳回第一列。

## 单元格内换行

在表格单元格内按 Enter 或 Shift+Enter 会插入换行。保存时 HorseMD 将其序列化为 `<br>`：

```md
| 姓名 | 说明 |
| --- | --- |
| 小马 | 第一行<br>第二行 |
```

GFM 表格的每一行必须保持为源码中的单行。使用 `<br>` 可以让内容重开后仍然正确解析，不会破坏后续行列。

## 源码模式编辑

源码模式中需要自行保持竖线数量和分隔行。单元格文字包含竖线时使用反斜杠转义；复杂内容建议切回富文本检查结果。

::: warning 不支持跨行跨列
标准 Markdown 表格没有 `rowspan` 和 `colspan`。需要复杂表格时可以使用受支持的原生 HTML 表格，但部分 Markdown 平台可能不会采用相同样式。
:::
