# 云同步文件夹

<span class="version-badge">适用于 HorseMD v0.7.0（桌面端）</span>

云同步只会处理你明确选择的文件夹。添加到左侧工作区不会自动上传；只有在“设置 > 云同步”中开启并连接云端后，HorseMD 才会准备同步计划。

## 开始前

- 准备一个 HTTPS WebDAV 地址，或一个 S3 兼容存储的 Endpoint、Bucket、Region、Access Key 和 Secret Access Key。
- 确认云端账号拥有该目录或 Bucket 的读取、写入、列举和删除权限。
- 第一次同步前建议保留重要文件的独立备份。同步会传播删除，不等同于自动备份。

密码和 S3 Secret Access Key 只保存在当前系统的安全凭据存储中，不会写入 Markdown、工作区文件夹或远端清单。

“测试并保存”和已保存连接旁的“测试连接”都会验证列举、上传和删除权限。验证时，HorseMD 会在 Bucket 的 `HorseMD/.connection-check/` 下短暂创建一个空测试对象后立即删除；这不会写入任何同步工作区。

## 开启一个文件夹的同步

1. 打开“设置”，进入“云同步”。
2. 在“同步文件夹”点击“添加要同步的文件夹”，选择真实磁盘目录。
3. HorseMD 只会在所选根目录创建隐藏的 `.horsemd/workspace.json` 身份标记；文件夹不会被移动，子目录也不会各自生成标记。
4. 在“云端连接”选择“WebDAV”或“S3 兼容存储”卡片，填写对应信息并点击“测试并保存”。两个连接入口彼此独立，可以同时添加多个不同服务的连接。
5. 回到该文件夹，选择连接后点击“连接并继续”。若云端还没有 HorseMD 工作区，会直接进入首次同步预览。
6. 若该连接中已经有工作区，才会出现“加入已有工作区”与“创建独立工作区”的选择，避免把本地文件误同步到另一份笔记。
7. 已连接的文件夹右侧提供图标操作：循环箭头为“双向同步”，上箭头为“上传本地到云端”，下箭头为“从云端下载到本地”，列表图标为“查看预览”。悬停图标可查看说明。
8. 第一次把本机文件夹放到空云端时，选择“上传本地到云端”；在另一台设备接收已有笔记时，选择“从云端下载到本地”。这两个方向化操作会在当前文件夹下方展开文件清单，确认后才传输；清单限制高度，可滚动查看全部文件。
9. 完成一次初始化后，日常使用“**双向同步**”：仅本地改动会上传，仅云端改动会下载；同一文件两边都改时保留冲突副本，不静默覆盖。同步进行时显示旋转图标，完成后显示勾选图标。

如果你在云服务后台手工清空了 HorseMD 工作区，HorseMD 会停止双向同步，而不会把空云端当成“删除本地文件”的命令。此时选择“上传本地到云端”即可用本机内容恢复云端；只有你明确选择“从云端下载到本地”时，才会以云端内容替换本地文件，旧文件仍会移入 `.horsemd/trash/`。

已保存的连接可点击“编辑”查看和修改地址、Bucket、Region、账号等信息。密码和 Secret Access Key 不会回显；留空表示继续使用系统安全存储中原有的值，重新填写才会替换。

远端为空时，首次同步会上传这个文件夹中的 Markdown、图片和普通附件。`.horsemd`、`.git`、`node_modules` 等内部目录不会作为用户内容上传。

## 云端连接怎么填写

HorseMD 会直接在设置页每个字段下显示示例。下面的内容用于帮助你从云服务商后台找到对应信息；示例地址、账号和密钥不能直接使用。

### WebDAV

适用于 Nextcloud、群晖 NAS、坚果云以及其他提供 WebDAV 的服务。

| 字段 | 填写内容 | 示例 |
| --- | --- | --- |
| 连接名称 | 仅用于 HorseMD 内区分连接，可自行命名 | `我的 Nextcloud` |
| WebDAV 地址 | 服务商提供的完整 WebDAV 根地址，通常以 `/` 结尾 | `https://cloud.example.com/remote.php/dav/files/alex/HorseMD/` |
| 用户名 | 服务商账号或 WebDAV 用户名 | `alex` |
| 密码或应用专用密码 | 服务商后台生成的应用专用密码优先，不建议填网页登录密码 | `abcd efgh ijkl mnop` |

不同服务的地址格式不同，应以服务商后台给出的 WebDAV 地址为准。地址不能填写普通网页地址，例如 `https://cloud.example.com/` 通常不是有效的 WebDAV 地址。

### S3 兼容存储

适用于 AWS S3、Cloudflare R2、MinIO 和其他 S3 API 兼容服务。

| 字段 | 填写内容 | 示例 |
| --- | --- | --- |
| 连接名称 | 仅用于 HorseMD 内区分连接，可自行命名 | `Cloudflare R2` |
| S3 Endpoint | 服务商提供的 API Endpoint，不是 Bucket 的网页地址 | `https://<账户 ID>.r2.cloudflarestorage.com` |
| Bucket 名称 | 预先在服务商后台创建的 Bucket | `horsemd-sync` |
| Region | Bucket 所在区域；R2 通常填写 `auto`，AWS 例如 `us-east-1` | `auto` |
| Access Key ID | 在服务商后台创建 API Token 后得到的访问密钥 | `AKIA...` |
| Secret Access Key | 与 Access Key ID 配套的私有密钥 | `...` |

不要把 Access Key 或 Secret Access Key 写入 Markdown 文件、截图或分享给他人。HorseMD 将这些敏感信息保存在当前系统的安全凭据存储中。

## 远端目录结构

每个同步文件夹在远端使用一个 UUID 目录，避免两个同名本地文件夹相互覆盖：

```text
HorseMD/
  <工作区 UUID>/
    文章.md
    assets/
    .horsemd/
      manifest.json
```

文章和附件直接保留原有相对路径；`.horsemd/` 仅保存同步清单和回收内容。你无需手动创建、修改或移动这些目录。

## 在另一台设备加入已有文件夹

若另一台设备已经同步过同一份笔记：

1. 在新设备上选择一个准备接收文件的本地文件夹，并在“云同步”中开启它。
2. 添加同一个 WebDAV 或 S3 连接。
3. 为该文件夹选择连接，点击“加入已有云端工作区”。
4. 选择列出的工作区。HorseMD 会先显示下载、上传和冲突预览。
5. 确认预览后开始同步。

不要在两台设备上把两个不同的本地文件夹同时当作同一个远端工作区的独立写入源。HorseMD 会阻止当前设备上的重复工作区登记，但跨设备仍应按“加入已有工作区”的流程操作。

## 冲突和删除

同一个文件在两端都修改且内容不同，HorseMD 不会静默选择一边覆盖另一边。它会保留双方内容，并创建带“冲突副本”字样的普通文件。

一端删除、另一端未修改时，删除会同步到另一端。远端删除内容会先进入 HorseMD 的远端回收路径；本地被同步删除的文件会移动到所选文件夹下隐藏的 `.horsemd/trash/`。在确认同步预览前，先检查删除数量是否符合预期。

## 当前范围

- 当前提供桌面端的手动“立即同步”。自动同步、历史恢复和移动端同步会在后续版本开放。
- 默认只接受 HTTPS 连接。局域网 HTTP、端到端加密和多个远端目标不属于当前功能范围。
- “停止管理”只取消本机的同步登记，不会删除本地文件、远端文件或根目录中的身份标记。
